    Introduction

    Unreal Engine is a powerful 3D creation platform developed by Epic Games. It is used in Game Development, Film Production, Architecture vizualization, Automative design, and Virtual reality experiences.

    It supports both visual scripting and text-based coding with languages like C++ and C#, and also provides tools for high-end graphics rendering, physics simulation, artificial intelligence, animation, and networking.

    Unreal Engine was created in 1998 with 5 versions of Unreal Engine releasing subsequently:
    • Unreal Engine 1:The original version released in 1988.
    • Unreal Engine 4:Released in 2014 with sub versions(4.0.1-4.27.0).
    • Unreal Engine 5:The current version, released in April 2022.
    • Unreal Engine 5.x:Sub-versions of Unreal Engine 5(5.0-5.6).
    • Unreal Engine 6:The next planned version of Unreal Engine which is still in development.

    Getting Started

    After installing Unreal Engine, there are two ways that the user can create projects:

    1. Blueprints:

    Blueprints are visual scripting systems that allow users to create game logic and functionality without writing traditoinal code(C++).

    2. C++:

    For C++ projects you will need a code editor and toolchain(a set of software tools used to compile, link, and build C++ programs

    Install Visual Studio 2022 with the following workloads:

    • Desktop Development with C++
    • Game Development with C++
    3. Unreal Engine Interface Components:
    • Viewport:Where you see and manipulate your 3D world
    • Content Browser:File explorer for assets like textures,meshes, and sounds
    • World Outliner:List of all actors(objects) in the level
    • Details Panel:Shows eligible properties for selected objects
    • Modes/Toolebar:Tools for placing objects,scuplting terrain, editing materials, and more.

    Core concepts

    1. Actors & Components

    Actors are the base class for all objects that can be placed in a level. Components are pieces of functionality that define what the Actor can do or how it behaves.

    Actors: Actors are objects in the game world.

  • Examples: characters, lights ,triggers, cameras, and pickups

    • Components:

  • Reusable pieces of functionality added to Actors

    • Common types:

    • StaticMeshComponent: displays 3D models.
    • BoxCollisionComponent: defines a collision box.
    • AudioComponent: plays sounds.
    • CameraComponent: attaches a camera to an actor.

    Every Actor has a Root Component that other components are attached to.

    2. Levels & Blueprints

    Levels: A level(also called a map) holds Actors, terrain, lights, and more. You can create multiple levels for different parts of the game of different gameplay states

    Blueprints: Blueprints are Unreal Engine's visual programming system, used for logic, behavior, and interaction with the use of C++

    Blueprint Types:

    • Actor Blueprints: add behavior to Actors.
    • Function Libraries: reusable Blueprint functions.

    Key Blueprint Elements:

    • Nodes: Represent actions, fucntions, or data.
    • Variables:Store data(integers, booleans, vectors, and more)
    • Events:BeginPlay, Tick, Overlap events,etc.

    Common uses

    • Player Movement
    • UI logic
    • Trigger zones
    • Ai behavior

    Scripting/Programming

    1. Blueprint System Overview

    Key Features:

    • Drag-and-drop node based system.
    • Event-drive(BeginPlay.etc)
    • Real-time feedback with easy debugging.
    • Great for logic, animations, UI, and AI.

    The pros of blueprint are the fast iterations meaning there is no compile time, and it is easier for beginners to learn.

    A Blueprint might detect when a player enters a trigger box,play a sound, display a message, and start a countdown timer for an event.

    2. C++ API Usage

    Unreal Engine's core is written in C++, and advanced gameplay systems often require or benefit from writing C++ code.

    With C++ you can create custom Actor classes, override engine bahavior, access low-level systems, and optimize performace.

    To benefit from both Blueprints and C++, use C++ for complex systems and core mechanics, and use Blueprints for specific interactions, animations, and visual logic.

    User Interface(UI)

    UMG is the primary toolset in Unreal Engine for building 2D UI. It lets you visually design layouts and bind them to date or events

    Some common Widgets include:

    • Text
    • Button
    • Image
    • Progress Bar
    • Canvas Panel
    • Horizontal/Vertical Box.
    1. UI styling and Animation

    UMG also allows styling and animation to create clean and responsize interfaces

    Styling options include: font, color, padding, etc.

    Animations inclue: fade-ins, buttons pulses, etc.

    Animation

    Unreal Engine's animation system is very powerful. It supports basic character movement, and complex cinematics. It also integrates tools from both real-time gameplay and pre-rendered scenes.

    1. Skeletons and Skeletal Meshes

    A Skeletal Mesh is a 3D model with a hierarchy of bones, it is used for animating characters, creatures, and anything that needs to move organically.

    2. Animation Blueprints

    Animation blueprints also known as AnimBps handle the logic that drives animations during gameplay.

    Purpose

  • Update character animations based on input, movement or custom conditions.

    • Key Component

  • Variables:Speed, isInAir, isCrouching, etc. Used to determine which animations to play.

    • Example:

    If character Speed > 0, play the running animation

    If isInAir = True, play the jump animation

    3. Animation tools

    Animation Montage: A tool for playing short, controlled animations like punches, reloading, or emotes.

    Notify Tracks: Trigger events during an animations, like spawning effects, playing sounds, or dealing damage.

    Physics and Collsions

    1. Physics Simulation

    The physics system enables dynamimc behaviors such as gravity, friction, and rigid body motion

    Key Properties:

    Mass: how heavy the object is.

    Linear/Angular Damping:Resistance to movement and rotation.

    Physics Material: Controls friction, bounce, etc.

    Common uses:

    Objects that fall, roll, or get thrown

    Ragdoll physics for characters

    Destructable environments

    2. Character Movement and Physics

    Characters in Unreal use a Character Movement Component which handles physics like behavior with extra game friendly features.

    Character Movement include: Walking, jumping, falling, gravity, ground detection, sliding and stepping up.

    Audio

    Unreal Engine has a flexible audio system that supports everything from simple sound effects to complex interactive soundscapes(audio built opon eachother).

    1. Sound Assets in Unreal

    Sound Wave:

    A .wav file is imported into the game engine representing raw sound date like voice lines, footsteps, music, and more.

    Sound Cues:

    A node based graph system to combine and modify Sound Waves, supporting random playback, delays, volume/pitch, and looping.

    EX:

    A Sound Cue for gunshots might randomly pick between 3 different shots and apply pitch variation so it doesn't sound repetitive.

    2. Spacial Audio(3D Sound)

    Spacial Audio allows you to place sounds in a 3D space so players can percieve its direction and distance.

    Attenuation, concurrency, doppler, and occlusion are settings used to control volume falloff, limit overlapping sounds, and simulate real world sound effects.

    3. Examples of Audio Playback

    Play sound at location - one time 3D sound in the world

    Spawn sound attached - sound moves with an Actor

    4. Interactive Audio

    Examples:

    • Footstep sounds based on surface type.
    • Music that changes when enemies are nearby.
    • UI sounds for clicks, hovers, or feedback.

    Multiplayer and Networking

    Unreal Engine has built in support for multiplayer gamers using a client-server architecture, making it easier to handle replication, latency, and authority in real-time games like shooters, RPGs, or co-op experiences.

    1. Core Concepts

    Server: Owns the truth of the game state and processes authoritative logic

    Clients: The player's device which sends input and recieve replicated date

    Authority: The server has authority over most gameplay decisions(movement, damage, spawns, etc.). Clients can request actions, but only the server can finalize them.

    2. Local Vs Online Multiplayer

    Listen Server: One player hosts and plays at the same time

    Dedicated Server: Represents a pure server(a server that enforces strict rules on client-side modificatoins), players connect to this remotely.

    3. Online Subsystems
    • Steam
    • Epic Online Services
    • PlayStation Netwoork
    • Xbox Live

    Troubleshooting

    Troubleshooting is about identifying, isolating, and fixing issues efficiently, whether they involve crashes, performance drops, visual glitches, or multiplayer bugs.

    1. Common Problems and Fixes

    Engine Crashes:

    Check Crash Logs: Usually saved in the Project Folder

    Update Drivers, especially graphics drivers.

    Use the crash reporter: This sends data to Epic and gives clues on the cause

    2. Debugging Tools

    The Output Log shows warnings, and errors. The Visual Logger records actor movement, collisions, and AI decision over time. Breakpoints can be used in both blueprints and C++ to pause and inspect game state in real-time.
